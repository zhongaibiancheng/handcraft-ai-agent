遇到问题？按顺序检查以下清单，90% 的问题都能解决。

## 问题1：模型完全无视我的 System Prompt

### 🔍 症状
- 设定了角色，但回复风格完全不像
- 让它简洁它还是啰嗦
- 让它用中文它还是输出英文

### ✅ 排查步骤（按顺序执行）

**Step 1 — 检查 Prompt 是否真的传进去了**

```python
# ❌ 常见错误：messages 里没有 system role
response = client.chat.completions.create(
    model="qwen2.5:7b",
    messages=[
        {"role": "user", "content": "你好"}  # 没有 system！
    ]
)

# ✅ 正确写法
response = client.chat.completions.create(
    model="qwen2.5:7b",
    messages=[
        {"role": "system", "content": "你是Python老师"},  # ← 确保有这行
        {"role": "user", "content": "你好"}
    ]
)
```

**Step 2 — 检查 Prompt 是否太模糊**

| ❌ 太模糊 | ✅ 更具体 |
|----------|---------|
| 你专业一点 | 用学术语言回答，每段引用至少一个技术概念 |
| 回答简短 | 回答控制在3句话以内，不要展开 |
| 用中文 | 全程使用中文回答，不要出现任何英文单词 |

**Step 3 — 检查指令语气是否太弱**

| ❌ 弱语气 | ✅ 强语气 |
|----------|---------|
| 请尽量... | **你必须**... |
| 可以考虑... | **务必**... |
| 尝试使用... | **严格使用**，**禁止**... |

**Step 4 — 检查模型能力**

不是所有模型都能很好地遵循复杂指令：

| 模型 | 指令遵循能力 | 建议 |
|------|:---:|------|
| Qwen2.5:7B | ⭐⭐⭐⭐ 好 | 本教程默认选择 |
| Llama 3.1 8B | ⭐⭐⭐⭐ 好 | 英文场景更强 |
| Qwen2.5:3B | ⭐⭐⭐ 一般 | 小模型能力有限，Prompt 要更简单 |
| 原始 Llama 2 | ⭐⭐ 较差 | 不推荐用于需要强指令跟随的场景 |

## 问题2：每次回复风格都不一致

### 🔍 症状
- 同一个问题问两次，回复长度差很多
- 有时正式有时随意
- 格式忽左忽右

### ✅ 解决方案

**方案 A：降低 temperature**

```python
# 默认值偏高 → 随机性大
temperature = 0.7

# 改为低值 → 更稳定一致
temperature = 0.3   # 推荐：0.2 ~ 0.4
```

> temperature 越低，模型的回复越"保守"，越倾向于选最高概率的词。创造性会降低但一致性更高。
>
> **实用取值参考：**
>
> | 值 | 效果 | 适用场景 |
> |:--:|------|---------|
> | 0.0 ~ 0.3 | 几乎确定性输出，每次结果高度一致 | 数据提取、格式化输出、代码生成 |
> | **0.3 ~ 0.5** | **轻微随机性，兼顾一致性和自然度** | **✅ 本教程推荐范围** |
> | 0.7 ~ 1.0 | 变化明显，回复更有"个性" | 创意写作、头脑风暴 |
> | > 1.0 | 输出发散，可能不连贯或跑题 | 一般不建议使用 |
>
> 注：API 定义的理论范围为 0~2，但超过 1.0 后实用性急剧下降，日常开发基本不会用到。

**方案 B：在 Prompt 中加硬性格式约束**

```
你是一位技术写作助手。

【格式要求 - 必须严格遵守】
- 第一段：一句话直接答案
- 第二段：详细解释（2-3句话）
- 第三段：一个具体示例
- 总字数不超过150字
```

**方案 C（进阶）：第三课的结构化输出**

用 JSON Schema 强制约束输出格式，彻底解决一致性问题。敬请期待第三课！

## 问题3：System Prompt 太长/太短

### 📏 长度参考指南

| 字数范围 | 效果评价 | 建议 |
|---------|:---:|------|
| < 50字 | ⚠️ 约束力不足 | 只适合简单角色切换 |
| **50 ~ 200字** | ✅ **最佳区间** | 推荐目标范围 |
| 200 ~ 500字 | ⚠️ 开始冗余 | 模型可能"注意力分散" |
| > 500字 | 🔴 过长 | 严重占用上下文窗口 |

### 💡 长度优化技巧

如果觉得内容太多塞不下：
1. **分优先级** — 把最重要的约束放前面
2. **用结构化格式** — 列表比段落更省 token 且更易被遵循
3. **拆分为多步** — 复杂任务拆成多个 Agent 各司其职

## 问题4：Ollama 连接报错

### 错误信息：`Connection refused` 或 `Connection error`

**原因：** Ollama 服务没有运行

```bash
# 终端1：启动 Ollama 服务（保持运行）
ollama serve

# 终端2：验证服务是否正常
curl http://localhost:11434/api/tags
```

Windows 用户注意：如果 `ollama serve` 报错端口被占用，可能是 Ollama 后台已经在运行了（安装后默认开机自启）。直接跑代码就行。

## 问题5：内存不足 / 速度很慢

### 症状
- 运行后长时间无响应
- 系统卡顿
- 报 `CUDA out of memory` 或类似错误

### 根据原因：模型太大，硬件扛不住

| 模型 | 内存需求 | 显存需求（GPU加速） | 速度体验 |
|------|:---:|:---:|:---:|
| qwen2.5:3b | 4GB | 3GB | ⚡ 快 |
| **qwen2.5:7b** | **8GB** | **6GB** | ✅ 中等 |
| qwen2.5:14b | 16GB | 10GB | 🐢 较慢 |
| qwen2.5:72b | 48GB | 40GB | 🔴 很慢或无法运行 |

**解决方案：**
1. 换小模型：`ollama pull qwen2.5:3b`
2. 关闭其他占用内存的程序
3. 如果有 GPU 但没用到，检查 Ollama 的 GPU 配置

## 问题6：中文输出质量差 / 出现乱码

### 可能原因及修复

| 原因 | 修复方式 |
|------|---------|
| System Prompt 是英文但想用中文对话 | 在 Prompt 末尾加："请全程使用中文回复。" |
| 模型本身中文能力弱 | Qwen 系列是中文原生模型，换 Qwen 即可 |
| 终端编码问题 | Windows PowerShell 执行：`chcp 65001` |

## 🚨 快速诊断清单

遇到问题时，按这个顺序逐项检查：

- [ ] 1. Ollama 服务在运行吗？（`ollama list` 能列出模型）
- [ ] 2. messages 参数里包含 system 角色了吗？
- [ ] 3. base_url 是 `http://localhost:11434/v1` 吗？
- [ ] 4. System Prompt 写得足够具体吗？（不是"专业一点"这种模糊表述）
- [ ] 5. 使用了强语气词吗？（必须、务必、严格、禁止）
- [ ] 6. temperature 设置合理吗？（0.3~0.5 提高一致性）
- [ ] 7. Prompt 长度在 200 字以内吗？
- [ ] 8. 模型和硬件匹配吗？（7B 至少要 8GB 内存）

全部打勾了还是有问题？那大概率是模型本身的概率性导致的偶发情况 — **再试一次就好** 😄
